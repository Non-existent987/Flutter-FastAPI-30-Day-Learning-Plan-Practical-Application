### Day 2: 数据模型 (Pydantic & Data Shapes)  
  
> **目标**：摆脱手动的 JSON 解析，学会使用 Pydantic 定义数据“长什么样”。这是 FastAPI 最强大的核心功能之一。  
  
#### 1. 核心概念：Pydantic  
  
FastAPI 不像 Flask 那样需要你自己去解析 `request.json()`。你只需要定义一个 Python 类，FastAPI 会自动完成以下工作：  
1.  **读取请求体**：从 JSON 中读取数据。  
2.  **数据校验**：确保 `id` 是数字，`title` 是字符串。如果类型不对，自动报错。  
3.  **自动文档**：根据你的类，自动在 `/docs` 页面生成交互式文档。  
  
#### 2. 准备工作  
  
虽然安装 `fastapi` 时通常会自动安装 `pydantic`，但为了保险起见（以及后续可能用到特定功能）：  
  
```bash  
pip install pydantic  
```
#### 3. 任务：定义 Article 模型
我们需要一个模型来代表你的“教程文章”。

代码实现 (main.py)
将以下代码添加到你的 main.py 中（或者覆盖昨天的练习代码）：

```python

from fastapi import FastAPI  
from pydantic import BaseModel # 1. 导入 Pydantic 的基类  
  
app = FastAPI()  
  
# 2. 定义数据模型  
# 这个类定义了前端必须发给你的数据格式  
class Article(BaseModel):  
    id: int  
    title: str  
    content: str  
    # 可选：如果要定义可选字段，可以使用 Optional[str] = None  
  
# 3. 创建一个接口使用这个模型  
@app.post("/articles/")  
async def create_article(article: Article):  
    # 这里的 article 参数已经是验证过的 Article 对象了  
    # 可以直接使用 article.title, article.content  
      
    # 模拟：我们只是把接收到的数据原样返回，证明解析成功  
    return {  
        "message": "文章接收成功",  
        "data": article  
    }  
```

#### 5. 学习总结 (Key Takeaways)
 Pydantic Model: 所有的请求体数据结构都应该继承自 BaseModel。  
 Type Hints: Python 的类型提示 (: int, : str) 在 FastAPI 中不仅仅是注释，它们决定了数据验证逻辑。  
 自动化: 定义好模型后，文档和验证全自动生成，无需手写 if not title: return error。
